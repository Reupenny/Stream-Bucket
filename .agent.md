This file is updated and maintained by agents as they work on the codebase. It contains living documentation about the project's structure, conventions, and important patterns.

## Project Purpose

Stream Bucket is a macOS application for batch processing video files into HLS streams with optional S3 upload and Cloudflare CDN integration.

## Key Patterns

### State Management

- `ProcessorState` is the single source of truth (ObservableObject)
- All `@Published` properties must be modified on MainActor
- Use `@EnvironmentObject` to share state across views

### Build Process

```bash
./build.sh [version]  # Recommended
# or manual:
swiftc -parse-as-library -target arm64-apple-macosx13.0 -O Sources/*.swift -o "Stream Bucket.app/Contents/MacOS/Stream Bucket"
```

### FFmpeg Integration

- `FFmpegWrapper.shared` is the singleton entry point
- Always check `checkFFmpeg()` before processing
- Use `run(arguments:onLog:)` for async execution

### S3 Integration

- Credentials stored in Keychain via `KeychainHelper`
- `S3Client` handles AWS Signature V4 signing
- Supports S3-compatible providers (Backblaze B2, AWS S3, Cloudflare R2)

## No Band-Aids Policy

**Fix problems at the source, not around them.**

- Rename modules, files, types, functions when cleaner names emerge
- Move responsibilities between layers when architecture improves
- Delete dead code, unused exports, and "just in case" parameters
- Do NOT leave TODO/FIXME notes ‚Äî fix the issue now
- Do NOT add adapters to hide wrong abstractions

## Common Tasks

### Adding an Encoding Preset

1. Add case to `EncodingPreset` enum in `ProcessorState.swift`
2. Add FFmpeg arguments in `BatchProcessor.swift`

### Adding a Resolution Option

1. Add `@AppStorage` property in `ProcessorState.swift`
2. Add UI toggle in `ContentView.swifa aadpueretdpu`.
3. Add resolution filter in `BatchProcessor.swift`

### Adding S3 Provider Support

1. Update endpoint handling in `S3Client.swift`
2. Add documentation in `SETUP.md`

## Debugging Checklist

1. **FFmpeg not found**: Check `FFmpegWrapper.findFFmpeg()` returns valid path
2. **S3 upload fails**: Verify Keychain credentials, check endpoint URL format
3. **Live stream issues**: Ensure port 1935 available, verify RTMP URL
4. **UI not updating**: Confirm `@Published` changes on MainActor
